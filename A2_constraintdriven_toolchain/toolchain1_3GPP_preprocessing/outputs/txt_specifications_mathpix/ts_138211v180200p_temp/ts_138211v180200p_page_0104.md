6.4.1.2.2 Mapping to physical resources
6.4.1.2.2.1 Precoding and mapping to physical resources if transform precoding is not enabled

The UE shall transmit phase-tracking reference signals only in the resource blocks used for the PUSCH, and only if the procedure in [6, TS 38.214] indicates that phase-tracking reference signals are being used.

The PUSCH PT-RS shall be mapped to resource elements according to
- if the higher-layer parameter dmrs-TypeEnh is configured
\[
\begin{array}{l}
{\left[\begin{array}{c}
a_{k, l}^{\left(p_{o l}, \mu\right)} \\
\vdots \\
a_{k, l}^{\left(p_{\rho-1}, \mu\right)}
\end{array}\right]=\beta_{\mathrm{PT}-\mathrm{RS}} W\left[\begin{array}{c}
r^{\left(\tilde{p}_{0}\right)}\left(4 n+k^{\prime}\right) \\
\vdots \\
r^{\left(\tilde{p}_{v-1}\right)}\left(4 n+k^{\prime}\right)
\end{array}\right]} \\
k=\left\{\begin{array}{ll}
8 n+2 k^{\prime}+\Delta & \text { configuration type } 1 \\
12 n+k^{\prime}+\Delta & \text { configuration type } 2, k^{\prime} \in\{0,1\} \\
12 n+k^{\prime}+\Delta+4 & \text { configuration type } 2, k^{\prime} \in\{2,3\}
\end{array}\right.
\end{array}
\]
- otherwise
\[
\begin{array}{c}
{\left[\begin{array}{c}
a_{k, l}^{\left(p_{o, l}\right)} \\
\vdots \\
a_{k, l}^{\left(p_{\rho-1}, \mu\right)}
\end{array}\right]=\beta_{\mathrm{PT}-\mathrm{RS}} W\left[\begin{array}{c}
r^{\left(\tilde{p}_{0}\right)}\left(2 n+k^{\prime}\right) \\
\vdots \\
r^{\left(\tilde{p}_{v-1}\right)}\left(2 n+k^{\prime}\right)
\end{array}\right]} \\
k=\left\{\begin{array}{cl}
4 n+2 k^{\prime}+\Delta & \text { configuration type 1 } \\
6 n+k^{\prime}+\Delta & \text { configuration type 2 }
\end{array}\right.
\end{array}
\]
when all the following conditions are fulfilled
- \( l \) is within the OFDM symbols allocated for the PUSCH transmission
- resource element ( \( k, l \) ) is not used for DM-RS
- \( \quad k^{\prime} \) and \( \Delta \) correspond to \( \tilde{p}_{0}, \ldots, \tilde{p}_{v-1} \)

The quantities \( k^{\prime} \) and \( \Delta \) are given by Tables 6.4.1.1.3-1 and 6.4.1.1.3-2, the configuration type is given by the higherlayer parameter dmrs-Type in the DMRS-UplinkConfig IE, and the precoding matrix \( W \) is given by clause 6.3.1.5. The quantity \( \beta_{\text {PTRS }} \) is an amplitude scaling factor to conform with the transmit power specified in clause 6.2 .2 of \( [6 \), TS 38.214].
